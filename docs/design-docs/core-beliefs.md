# Core Beliefs

1. A mock server is useful only when its success and failure modes are explicit.
2. Public administration can be acceptable for a test service when destructive operations are lightly but consistently protected.
3. Endpoint permissions must be visible in OAuth consent and mechanically enforced in runtime calls.
4. Every protocol behavior should be proven by a unit or E2E check before Ralph promotion.
5. Operators need reset, logs, health, and startup proof as much as feature code.
